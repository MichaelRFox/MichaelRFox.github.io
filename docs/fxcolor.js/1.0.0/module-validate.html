(inner) validate(input, keys, ranges, parser) → {Object}

Source:
  • utility/validate.js, line 66

Takes an input to be used to set the fxColor class object'sfxColor color, validates the input and returns the components and alpha value.

Parameters:
Name Type Description
input Array | Object | String

The input used to set the color. If it is an array it should contain one element for each of the color components (e.g., three for an RGB color or four for a CMYK color). If it is an object it must have key/value pairs for each of the color components (e.g. {r: 255, g: 128, b: 0}). If it is a string, it must be a properly formmated CSS color specification (e.g., 'rgb(255, 128, 0)'. Optionally an additional value for the alpha chabbel may be provides for each. Inthe case of an object the key must be 'alpha'.
keys Array

An array of the keys for the color specification (e.g., ['r', 'g', 'b']). The alpha key must not be incuded.
ranges Array

An array of objects with key/value pairs containing the minimum and maximum allowable values for each color component to include the alpha channel (e.g., [{min:0, max:255}, {min:0, max:255}, {min:0, max:255}, {min:0, max:1}]).
parser function

The parse function to be used in the case of string input (e.g., parseRgb).
Returns:

An object containing key/value pairs for the components (an object with key/value pairs for the color components (e.g., {r: 255, g: 128, b: 0})) and the alpha value. Either one of these elements may be undefined if either the input is invalid or the alpha chanel value was not provided.

Type
Object
